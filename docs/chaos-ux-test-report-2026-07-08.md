## 详细发现

### 1. 连点提交无防护（F-01 / F-05 / F-15）

**现象**
- Sign In、Filter、Save Product 均可快速连点
- 登录连点触发 5 次 `/api/auth/login` 请求（控制台 401 刷屏）
- Filter / Save 无 loading 态锁定

**用户感受**
- 不知道系统是否在处理
- 可能重复创建产品或重复打 API

**建议**
- 提交后 `disabled` + debounce
- 登录/注册按钮增加 `loginLoading` 全局锁（登录有部分，Filter 没有）

**归类**：【前端Bug】

---

### 2. 搜索框超长输入无边界提示（F-04）

**现象**
- 5000 字符 + `<>\"';--{}` 等特殊符号可输入并提交
- 页面不白屏，但无「已截断/过长/无结果原因」说明

**API**
- 后端接受长 keyword（需限制长度与日志监控）

**建议**
- 前端 `maxlength` + 字符计数
- 空结果时区分「无匹配」与「输入异常」

**归类**：【前端Bug】/ 体验

---

### 3. 注册/username 长度限制前后端不一致（F-11）

**现象**
- 前端可输入数千字符用户名
- 提交后 API 返回 400：`username must be at most 64 characters`
- 登录超长用户名则仅返回 401 `invalid username or password`（不提示长度）

**建议**
- 登录/注册表单统一 `maxlength=64`
- 错误信息透传后端 validation message

**归类**：【前端Bug】

---

### 4. 数字筛选项异常值无反馈（F-12）

**现象**
- Category ID 输入 `-999999` 后 Filter
- 表格 0 行，显示空状态，**无「请输入有效 Category ID」**

**建议**
- 前端 min=0 校验或后端 400 + 友好提示

**归类**：【前端Bug】/ 体验
